django-greenqueue

Alternativa ligera a Celery
Andrei Antoukh
@niwibe

¿Por que una alternativa?


  • Celery es muy grande y tiene muchas dependencias.
  • Configuración compleja cuando necesitas algo pequeño.
  • El codigo fuente de celery es bastante complejo y lioso.

"Ventajas":


  • Una sola dependencia inicial: zeromq
  • Código extensible para implementar backends.
    Actualmente: sync, zeromq y rabbitmq
  • Curva de aprendizaje muy ligera, sobretodo para los que vienen de Celery.
  • Muy bien integrado con gevent.
    Si se usa: zeromq + gevent.
  • Backend especifico para unittest: sync
  • Configuración inicial mínima.

¿Como se usa?

Settings

# settings.py
GREENQUEUE_TASK_MODULES = ['yourproject.yourapp.async_tasks']
INSTALLED_APPS += ['greenqueue']

Async tasks

# yourproject/yourapp/async_tasks.py
from greenqueue.core import Library
register = Library()

@register.task(name='sum')
def sum(x,y):
    return x+y

Views

# yourproject/yourapp/views.py
from django.views.generic import View
from django.http import HttpResponse
from greenqueue import send_task

class Home(View):
    def post(self, request):
        # ...
        async_result = send_task('sum', args=[2,3])
        return HttpResponse("Sum: {0}".format(async_result.wait()))

Run worker

python manage.py run_greenqueue -v2
